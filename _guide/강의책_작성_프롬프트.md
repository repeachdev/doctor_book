# 📋 강의책 자동 작성 프롬프트 (재사용용)

> **사용법** — 아래 `=== 프롬프트 시작 ===` 부터 `=== 프롬프트 끝 ===` 까지를 복사해서 새 세션에 붙여넣고, 맨 윗줄의 `{PDF 경로}` 만 이번에 작업할 교재 PDF 경로로 바꾸면 됩니다. (예: `C:\Users\Leadwin\Downloads\책\1\1. 순환기.pdf`)

---

=== 프롬프트 시작 ===

너는 타학원 강사다. 아래 교재 PDF를 기반으로, **각 중단원을 1개의 .md 파일로** 만드는 "강의책 초안"을 작성한다.

**작업 대상 PDF:** `{PDF 경로}`
**출력 위치:** PDF와 같은 폴더 (각 중단원 → `01.md`, `02.md`, … 식으로 0패딩 2자리)

이 작업은 이미 검증된 방식이다. 아래 절차와 규칙을 **하나도 빼지 말고** 그대로 따른다. 품질 기준은 "원서를 학습용으로 충실히 옮기되 강의 흐름으로 재구성"이다.

---

## STEP 0 — 환경 준비 & PDF 읽기 방식

- 이 교재 PDF들은 **텍스트 레이어가 깨져 있다**(한글 폰트 인코딩 문제로 `get_text()`가 깨진 글자를 뱉음). 따라서 **텍스트 추출에 의존하지 말고, 페이지를 이미지로 렌더링한 뒤 비전(Read)으로 직접 정독**한다.
- 렌더링은 PyMuPDF(`fitz`)를 쓴다. 파이썬에 설치돼 있다(`import fitz`). 없으면 사용자에게 알린다.
- 작은 글씨의 **표/진단기준/수치/약물목록**은 반드시 해당 영역을 고배율(`fitz.Matrix(4~5, 4~5)`)로 **크롭 확대**해서 글자를 확정한다. 추측 금지.
- **읽기 검증(예방)** — 질환명·약물명·수치처럼 **한 글자 차이로 뜻이 달라지는 항목**은 크롭 확대본에서 글자를 하나씩 확인한다. 흐릿해 단정 못 하면 추측하지 말고 `〔원문 재확인 필요〕`로 남긴다. (특히 **닮은 글자 주의**: 결핵/경색, 흉벽/폐벽 등 — 오독은 만들고 나서 잡기보다 만들지 않는 게 싸다.)
- 렌더 산출물(PNG)은 **최종물이 아니라 스크래치**다. **임시폴더 한 곳**(예: `{PDF 폴더}\_render_tmp`)만 쓰고, 여기저기 흩어지게 만들지 말 것. 작업 전체가 끝나면 STEP 3 마지막에 이 폴더를 **삭제**한다(필요하면 다시 렌더하면 됨).

PowerShell + 파이썬 예시(페이지 전체 렌더):
```python
import fitz, os
doc = fitz.open(r"{PDF 경로}")
out = r"C:\Users\Leadwin\Downloads\책\_work\pages"; os.makedirs(out, exist_ok=True)
for idx in range(doc.page_count):
    doc[idx].get_pixmap(matrix=fitz.Matrix(2.4,2.4)).save(os.path.join(out, f"doc{idx:02d}.png"))
```
크롭 예시:
```python
W,H = doc[0].rect.width, doc[0].rect.height
doc[IDX].get_pixmap(matrix=fitz.Matrix(5,5), clip=fitz.Rect(0, H*0.45, W, H)).save(...)
```

## STEP 1 — 단원 구조와 페이지 매핑 파악

1. 앞쪽 페이지(표지·목차·중단원 시작부)를 렌더링해 비전으로 읽고, **대단원 1개 + 중단원 N개 + 각 중단원의 페이지 범위**를 표로 정리한다.
2. **페이지 번호 기준을 확정한다(혼동 주의):**
   - 사용자/본문에서 쓰는 "인쇄본 N쪽" = **PDF를 1쪽부터 센 순서**다. (`doc 인덱스 = 인쇄쪽 − 1`)
   - 책 하단 **푸터에 찍힌 번호는 다를 수 있다**(이 PDF가 큰 교재의 발췌라 보통 +몇 쪽). 푸터번호로 표기하지 말 것.
   - 표지/백지/목차가 앞에 있으면 `doc 인덱스`와 `인쇄쪽` 오프셋이 생기니, 실제 페이지를 렌더해 **내용으로** 확인한다(헤딩 텍스트로 검증).
   - 푸터번호와 인쇄쪽의 차이(오프셋)를 계산해 둔다(읽을 때 푸터번호에 헷갈리지 않기 위함). 본문의 모든 `p?` 는 푸터번호가 아니라 **PDF 순서 쪽수**로 표기한다.
3. 확정한 "중단원 목록 + 페이지 범위 + 오프셋"을 사용자에게 1줄씩 보고한다(긴 설명 없이).
   - ⚠️ 사용자가 "중단원"이라 부르는 단위와 책의 소제목이 헷갈릴 수 있다. 책에서 큰 번호 뱃지(예: `01 심혈관계 진찰`)가 중단원, 그 안의 작은 번호 뱃지(예: `01 혈압측정`)가 소단원이다. 같은 이름이 중단원·소제목 양쪽에 나오면 소단원 이름을 내용에 맞게 살짝 바꿔 붙이고 그 사유를 본문에 한 줄 명시한다.
4. **공통 표기 규약 카드 확정 (중단원 간 일관성용)** — 병렬 작성 에이전트들이 제각각 흐르지 않도록, 아래 항목을 **한 번 정해 모든 브리프에 동일하게** 넣는다(골드샘플과 어긋나지 않게):
   - **소단원 넘버링 스타일** 1가지로 통일 (예: `### 1. 제목` — `### 01.`·`### 1)` 혼용 금지).
   - **영어 용어 표기**: 처음 나올 때 `영어(한국어)` 병기 후 이후 약어 — 정책 1개로.
   - **형광펜 농도 기준**: 빨간 형광펜·빨간 볼드밑줄은 중단원당 핵심 몇 개로 절제(남발 금지).
   - **요약/암기 박스·기출 태그 형식** 1가지로 통일 (예: `> 🧩 이것만은 꼭!`, `> 💡 [기출] …`).

## STEP 2 — 중단원 개수만큼 병렬 작업

> ### ★★★ 동시 발사 규칙 (이 작업의 핵심·위반 금지) ★★★
> 서브에이전트를 띄울 때는 **반드시 단일 응답(하나의 메시지) 안에서 해당 묶음의 N개 Agent 호출을 한꺼번에** 한다.
> **❌ 절대 금지:** 에이전트 1개만 띄우고 그 결과를 기다린 뒤 다음 1개를 띄우는 행위(= 사실상 순차 실행). 이건 병렬이 아니다.
> **❌ 절대 금지:** "중단원 하나 매핑 → 하나 띄우고 대기 → 다음 하나 매핑" 식의 번갈아 진행.
> 동시성 상한으로 한 번에 다 못 띄우면 **묶음(batch)으로 쪼개되, 각 묶음 내부는 반드시 한 메시지에서 동시 발사**하고 그 묶음이 끝나면 다음 묶음을 또 동시 발사한다.
> 이 규칙은 에이전트를 띄우는 **모든 지점(STEP 2 작성 / STEP 3-② 검수 / 재실행)에 똑같이** 적용된다.

- **선행 조건:** STEP 1에서 **모든 중단원의 페이지 범위 매핑을 전부 끝내고** 1줄씩 보고를 마친 뒤에 STEP 2를 시작한다. 매핑을 다 끝내기 전에는 어떤 작성 에이전트도 띄우지 않는다(매핑은 푸터 오프셋 한 번만 확정하면 전 중단원 범위가 함께 나오므로 한꺼번에 한다).
- 중단원이 N개면 **N개의 서브에이전트를 한 메시지에서 동시에 띄운다**(위 동시 발사 규칙). 각 서브에이전트는 **중단원 1개**를 맡아 `NN.md`를 직접 작성한다.
- **작업 전 선(先)숙지 지시** — 각 서브에이전트는 **본문을 한 글자라도 쓰기 전에** ① 아래 `## 작성 규칙` 전체와 ② 아래 **골드샘플**을 **먼저 읽고** 그 수준·형식을 기준으로 작성에 들어간다. 작성 규칙이 표기·위계·충실도의 **단일 기준**이고, 골드샘플은 "잘 된 강의책이 어떻게 생겼는가"의 본보기다.
  - **골드샘플(형식 본보기)**: `_guide\gold_sample\09.md`(암기틀·형광펜 규약·표), `_guide\gold_sample\12.md`(표 활용 종합) — *책 루트(`...\책\`) 기준 상대경로*. 이 둘의 **위계·강의 들어가며·형광펜 3종·표/암기틀/callout 활용 수준과 밀도**를 기준으로 삼는다. (보조: 같은 폴더 `01.md` 흐름 재배열, `06.md` 감별표.)
  - ⚠️ **골드샘플은 '형식·구성·완성도'의 본보기일 뿐, 의학 내용을 베끼는 대상이 아니다.** 내용은 어디까지나 네가 맡은 중단원의 원문만이 기준이다. (골드샘플 자체에도 사소한 오류가 있을 수 있으니 *그 표현·수치를 가져오지 말 것*.)
- **각 서브에이전트에게 넘길 브리프**에는 다음을 모두 포함한다:
  1. 맡을 중단원 이름 + 인쇄쪽 범위(+ doc 인덱스 범위) + 푸터 오프셋
  2. 출력 파일 경로(`{같은 폴더}\NN.md`)
  3. STEP 0의 렌더/크롭 방식(텍스트 레이어 깨짐 → 비전 정독, 표는 확대)
  4. 아래 ## 작성 규칙 전체 — **읽고 숙지 후 작성하라**고 명시(이것이 단일 기준)
  5. **골드샘플 경로(위 09.md·12.md)** — "형식·완성도 기준으로 먼저 읽되 내용은 베끼지 말라"고 명시
  6. **STEP 1-4에서 확정한 공통 표기 규약 카드** (넘버링·영어표기·형광펜 농도·박스/태그 형식) — 모든 에이전트에 동일하게
  7. 이미 추출된 그림이 `...\assets\{과목}\` 에 있으면 경로를 알려주고 `(확보:)`로 표기하게 함

## STEP 3 — 검수(QA) : ① 형식 체크 + ② 원문 대조 검수

검수는 **두 단계**다. ②가 핵심이며, ②는 글을 쓴 에이전트가 아니라 **독립된 검수 에이전트**가 해야 한다(쓴 사람은 자기가 지어낸 것을 인식하지 못한다).

**3-① 형식 체크 (오케스트레이터가 기계적으로)**
- [ ] `#`(대단원) → `##`(중단원) → `###`(소단원) 위계가 정확한가. 소단원을 다른 소단원 안에 넣지 않았는가.
- [ ] 강조 표기가 `(노란 형광펜)` `(빨간 형광펜)` `(빨간 볼드 밑줄)` 3종 규약을 지키는가(남발 없는가).
- [ ] 그림 callout이 `🖼 [그림필요 · p?] 설명 — (확보/미확보)` 형식인가, 페이지가 PDF순서 기준인가.

**3-② 원문 대조 검수 (중단원마다 독립 검수 에이전트 1명·필수)**
- ⚠️ **동시 발사 규칙 적용(STEP 2와 동일):** 3-①(형식 체크)은 에이전트 없이 오케스트레이터가 **전 중단원을 먼저 다 훑고**, 그 다음 **검수 에이전트 N개를 단일 응답에서 한꺼번에** 띄운다. **파일 하나 검수 띄우고 기다린 뒤 다음 것을 띄우는 식(번갈아 진행) 금지.**
- 검수 에이전트에게: 「검수할 `NN.md` 파일 경로 + 그 중단원의 원본 인쇄쪽 범위(doc 인덱스 범위) + STEP 0의 렌더 방식」을 준다.
- 검수 에이전트는 **글을 새로 쓰지 않는다**. 대신:
  1. 해당 중단원 원본 페이지를 **다시 렌더해서 비전(Read)으로 읽는다**(표·수치는 크롭 확대해 글자 확정).
  2. `NN.md`의 모든 주장·수치·목록·약물명을 원문과 **한 항목씩 대조**한다.
  3. 다음을 잡아 목록으로 보고한다:
     - **(A) 할루시네이션** : 원문에 없는 *의학적 사실/주장*이 추가됨 → 삭제 또는 `〔원문 재확인 필요〕`
     - **(B) 누락** : 원문의 *사실·수치·목록 항목·약물명*이 빠짐 → 보강(과압축 금지)
     - **(C) 오기** : 수치·단위·방향·좌우·질환명이 원문과 다름 → 수정
     - **(C-용어) 임의 용어 치환** : 질환명·증후군명·해부명·검사명·약물명·술기명·분류명을 원문과 **다른 단어(동의어·순화어·임의 번역)로 바꾼 것** → 원문 표기로 되돌림. (예: 원문 "심방세동"인데 "심방잔떨림"으로 적음 → 잡음.) 단, 원문 용어를 **그대로 두고** 괄호로 풀이만 덧붙인 건 통과.
     - **(C-그림) 그림 callout 페이지 오기** : 모든 `🖼` callout의 `p?`를 **원문에서 직접 한 건씩 확인**(끝부분 그림이 한 칸씩 밀리는 off-by-one이 잦다) → 틀리면 수정
     - **(B-표) 표의 이미지化 누락** : 원문의 표가 **본문 텍스트 표로 옮길 수 있는데** 이미지 callout으로만 대체됐으면 → 표 안 데이터(행·열·수치)가 본문에 없으므로 (B)로 플래그, 텍스트 표 전환 대상으로 표시
     - **(D) 재구성 품질 (권고·비차단)** : 사실 오류는 아니지만 *강의책으로서 덜 다듬어진 곳*을 개선 제안으로만 보고한다(작성규칙 G 기준). 예: 산문째 방치돼 표로 옮기면 좋을 곳 / 흐름이 원문 목차 복붙에 가까운 곳 / 강조·암기틀이 비어 밋밋한 곳. **이건 A/B/C 같은 필수 수정이 아니라 권고이며, 새 의학정보 추가를 제안해선 안 된다.**
  4. **구조 커버리지 체크(번호 항목 소실 점검)** : 원문의 번호 매겨진 항목을 **전부 나열**한 뒤, 각 항목의 *내용*이 `NN.md` 어딘가에 (이름이 바뀌거나 합쳐졌더라도) 살아있는지 1:1로 확인한다.
     - ⚠️ 이것은 **구조를 원문대로 되돌리라는 게 아니다.** 강의 흐름에 맞춘 재편·병합·재번호·이름변경·그림 흡수는 **허용(통과)**한다. 내용이 보존돼 있으면 손대지 않는다.
     - 오직 *항목의 내용이 실제로 사라진 경우*만 (B) 누락으로 잡는다.
  5. 불확실한 원문 글자는 임의로 판정하지 말고 `〔원문 재확인 필요〕`로 남긴다.
  6. **보고 증거 의무** (이 보고는 오케스트레이터용 — `NN.md`엔 절대 넣지 않는다. "원문 p? 발췌" 류 문구가 강의책 본문에 들어가면 안 됨):**
     - **커버리지**: 원문 번호항목 N개 중 M개 대조 완료(미대조가 있으면 그 항목을 명시).
     - **각 지적마다 원문·md를 나란히 인용** → `원문(p?): "…" / md: "…"` 형식. 오케스트레이터가 재렌더 없이 즉시 판단·수정할 수 있게 함.
     - 지적이 0건이면 "N/N 대조, 불일치 없음"으로 명시 보고(빈 보고 금지).

  **검수 강도 보정** — 기준은 두 갈래다: **(가) 용어·수치는 글자 대조**(원문 표기 고정), **(나) 설명·연결 흐름은 사실 대조**(표현 달라도 사실 같으면 통과).
  - **잡을 것:** 위 (C-용어)·(C)대로, *질환명·약물명·검사명·해부명·분류명을 동의어/순화어/임의번역으로 바꾼 것*, *수치·단위·방향을 바꾼 것* → (C)로 보고.
  - **잡지 말 것(오탐 금지) — 흐름 문장의 표현 차이:**
    - 설명·연결 문장의 동의 패러프레이즈/말 다듬기 (예: "팔 둘레에 따라 적절한 크기의 공기주머니를 사용" → "팔 둘레에 맞는 크기를 사용") — ✅ 통과 (단, 그 안의 **용어·수치 자체**는 원문과 같아야 함)
    - 한 문장을 불릿·표로 분해, 항목 순서 재배치, 군더더기·순수 반복 제거 — ✅ 통과
    - `> **강의 들어가며**` 등 흐름/암기 틀 문장 추가(새 의학사실 아님) — ✅ 통과
  - 보고 대상은 **사실·용어·수치가 추가·삭제·변경된 경우(A/B/C)뿐**이다. 흐름 문장의 단순 표현 차이는 보고하지 않는다.
- 검수 결과(A/B/C 목록)를 받아 오케스트레이터가 `NN.md`를 수정한다. 수정량이 크면 해당 작성 에이전트를 재실행한다.
  - ⚠️ **동시 발사 규칙 적용(STEP 2와 동일):** 재실행 대상 중단원이 여러 개면 그것들도 **단일 응답에서 한꺼번에** 재실행한다. 하나 재실행하고 기다린 뒤 다음 것을 띄우는 식 금지.
- **무할루시네이션 원칙:** 책에 없는 의학정보는 표준지식이라도 절대 추가 금지. 이 단계의 목적은 "강의책이 원서를 벗어나지 않았음"을 원본 대조로 **증명**하는 것이다.

**3-③ 정리** — 모든 검수·수정이 끝나면 STEP 0에서 만든 임시 렌더 폴더(`{PDF 폴더}\_render_tmp`)를 **삭제**한다. 최종 산출물은 `NN.md`들과 `assets/`만 남긴다.

## 작성 규칙 (각 중단원 .md 공통)

**A. 위계 / 구성**
- `#` 대단원(과목명) → `##` 중단원 → `###` 소단원. 책 인덱스를 따르되, 강의 흐름(진찰·증상→검사 순 등)에 맞게 소단원 순서/이름을 약간 재편해도 된다(내용은 보존).
- 각 중단원 맨 위(`##` 중단원 제목 다음)에 `> **강의 들어가며**` 1문단으로 그 단원의 흐름·출제 경향을 짧게 깔아준다(새 의학사실 추가 X, 틀만 제공).

**B. 충실도 / 무할루시네이션**
- 원서의 내용·세부·수치는 **빠짐없이** 옮긴다. 순수 반복만 제거(과압축 금지).
- **★핵심★ 용어·수치는 원문 표기 그대로 고정** — 아래 항목은 임의로 동의어·의역·풀어쓰기로 바꾸지 말고 **원교재에 쓰인 글자 그대로** 옮긴다:
  - **질환명·증후군명·해부명·검사명·약물명·술기명** (예: 원문이 "심방세동"이면 "심방잔떨림"으로 바꾸지 않음, "Torsades de Pointes"를 임의 번역하지 않음)
  - **모든 수치·단위·범위·방향(좌/우·상/하)·등급·분류명** (반올림·단위환산·근사치화 금지)
  - 원문 용어가 어려워도 임의 순화 금지. 풀이가 필요하면 원문 용어를 **그대로 두고** 괄호나 별도 설명으로 덧붙인다.
- **자유롭게 재구성 가능한 것은 연결·도입부·순서·암기 틀 같은 '엮는 방식'뿐이다.** 반면 원문의 **정의·기전·인과·분류 설명은 '내용'**이므로 의미를 바꾸거나 원문에 없는 비교·조건·항목을 더하지 말 것. 말투만 다듬되 뜻은 원문 그대로 둔다.
  - (예: "흉벽을 통해 음성 전파를 손으로 느끼는 것"이라는 정의를 "폐벽을 통해 양쪽이 비슷한지 진찰"로 바꾸면 → 용어 오기(흉벽→폐벽) + 정의 의미변경 + 없는 내용 추가(양쪽 비교) = 위반.)
  - 즉 경계는 **용어·수치·정의·기전 = 고정 / 순서·도입·묶음·암기틀 = 자유**.
- **책에 없는 의학 정보는 절대 추가하지 않는다**(표준지식이라도). 덧붙일 수 있는 건 강의 흐름/암기 틀/책에 있는 시험팁뿐.
- 원문 글자가 확실치 않으면 추측하지 말고 `〔원문 재확인 필요〕` 표기.
- **분량 = 원문 분량에 비례한다.** 모든 사실 항목을 보존하므로 4쪽짜리 중단원은 짧고 22쪽짜리는 길다 — 그게 정상이다. 중단원끼리 분량을 인위적으로 균일하게 맞추려고 늘리거나 줄이지 말 것. (사실 하나를 얼마나 풀어쓰는지의 *밀도*는 골드샘플 수준을 따른다.)

**C. 강조(형광펜) 표기** — 디자인 단계에서 그대로 칠하도록, `**볼드**` 바로 앞 괄호로 지시
- `(노란 형광펜)**…**` : 시험 빈출 핵심 키워드·수치
- `(빨간 형광펜)**…**` : 최빈출·감별의 결정적 포인트
- `(빨간 볼드 밑줄)**…**` : 통째로 암기할 진단기준·공식
- 남발 금지. 정말 중요한 것만.

**D. 그림 callout**
- 형식: `> 🖼 [그림필요 · p?] {무슨 그림인지 + 라벨/구성요소} — (확보: 경로)` 또는 `(미확보: 원서 p? 삽화 스캔 필요)`
- 표로 옮길 수 있는 표는 본문 표로 옮기고, 해부도·파형·알고리즘 등 그림 자체가 필요한 것만 callout.

**E. 시험팁**
- 책의 이탤릭/별표 시험 코멘트나 강의 강조점은 `> 💡 …` 인용구로 보존.

**F. 영어 용어**
- 영어-only 용어는 한글로 억지 번역하지 않고 영어 유지(필요 시 `English(한국어)` 병기). 약어는 처음 나올 때 풀어 쓴다.

**G. 좋은 재구성이란 (이 작업의 가치·필사 방지)**
- "충실히 옮긴다"가 "원문 목차·문장을 그대로 복붙한다"는 뜻이 **아니다.** 충실도는 *내용*에 대한 것이고, *배치·표현 틀*은 강의책답게 끌어올린다. 아래는 **새 의학정보 추가가 아니라, 원문 내용 안에서 구조·표현만 손보는** 것이다:
  - **임상 흐름 순 배열** : 흩어진 내용을 진찰→증상→검사→진단→치료 같은 강의 흐름으로 재배열.
  - **동일 주제 묶기** : 책 여기저기 흩어진 같은 주제를 한데 모음(내용 보존).
  - **산문 표를 본문 표로 전환** : 원문이 줄글로 나열한 분류·비교를 마크다운 표로(데이터 누락 없이).
  - **출제 빈출점 강조** : 형광펜 3종 규약으로 시험 포인트를 시각화(남발 금지).
  - **암기 틀 제시** : 통암기 대상에 두문자·비교표 같은 틀 제공(틀만, 새 사실 X).
- 판단 기준: "이게 원문을 그냥 옮긴 건가, **강의용으로 더 잘 정리한 건가**?" 후자가 되도록.

먼저 STEP 1까지 해서 중단원 목록·페이지 매핑·오프셋을 보고하고, 곧바로 STEP 2 병렬 작업에 들어가라.
